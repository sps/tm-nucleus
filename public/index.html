Color Function

The above colors are stored within a SASS map called $palette. To access a color, use our custom color function by passing in the color group and color name as parameters.

// scss
$caution-color: color('yellow', 'malibu');

// output
$caution-color: #fecb34;

Note that, in general, your variable name should be abstract and not include the name of a color (i.e., use $alert-color and not $alert-red).

All color declarations should be kept in _palette.scss. The only place you should be using the color function is in this file. We do this so that all color defintions are kept in a single place.

Typography

Responsive fonts

With few exceptions, font sizes on our pages should be responsive at certain breakpoints. You should assume that any font size specified by a designer should be responsive unless explicitly stated otherwise.

To simplify the process of creating responsive fonts, we have a responsive-font() mixin to generate the appropriate code when passed a font size.

Note that we actually specify our font sizes in rem units. 1rem is equivalent to 10px.

// scss
.sample-text {
  @include responsive-font('md');
}

// output
.sample-text {
  font-size: 1.4rem;
}

@media (min-width: 45em) {
  .sample-text {
    font-size: 1.6rem;
  }
}

Using this mixin, our font sizes are then generated as follows (up to the specified widths).

Extra small copy 10px, 12px @include responsive-font('xs');
Small copy 12px, 14px @include responsive-font('sm');
Medium copy 14px, 16px @include responsive-font('md');
Large copy 18px, 20px @include responsive-font('lg');
Extra large copy 18px, 20px, 24px @include responsive-font('xl');

Non-responsive fonts

When defining a fixed font size, use our fontsy() function.

fontsy() can accept two parameters. The first is required and specifies the font size you want. The second parameter is optional and specifies the "version" of the font size (base or plus).

If a second parameter is not passed, it defaults to calling the base font size.

Base
Extra small copy 10px fontsy('xs')
Small copy 12px fontsy('sm')
Medium copy 14px fontsy('md')
Large copy 18px fontsy('lg')
Extra large copy 18px fontsy('xl')
Plus
Extra small copy 12px fontsy('xs', 'plus')
Small copy 14px fontsy('sm', 'plus')
Medium copy 16px fontsy('md', 'plus')
Large copy 20px fontsy('lg', 'plus')
Extra large copy 20px fontsy('xl', 'plus')

For the xl font-size only, there is a third option of plus2.

Plus2
Extra large copy 24px fontsy('xl','plus2') 
// scss
.fontsy-example {
  font-size: fontsy('md','plus');
}

// output
.fontsy-example {
  font-size: 1.6rem;
}

Form Input Font Size

Because iOS zooms in on any input field whose font size is less than 16px, we set the minimum font size of input fields to 1.6rem via $min-input-font-size.

Note that font sizes for input fields are fixed and not responsive.

// scss
.input-field {
  font-size: $min-input-font-size
}

// output
.input-field {
  font-size: 1.6rem;
}

Line Height

Line height is standardized at 1.35, which is equivalent to 135% of the font size.

Mixins for text styling

In the case where any of the above use cases is not appropriate, there are mixins available to achieve the desired appearance.

hdg-text mixin

The hdg-text() mixin can be used to create heading text that's bold with a bottom margin of 5px. It is called by passing in a font-size and color. It will automatically make the font responsive via a built-in call to the responsive-font() mixin.

// scss
.some-heading {
  @include hdg-text('md', $black);
}

// output
.some-heading {
  font-weight: 700;
  letter-spacing: -0.025em;
  font-size: 14px;
  margin-bottom: 5px;
  color: #000;
}

@media (min-width: 45em) {
  .some-heading {
    font-size: 16px;
  }
}

alt-copy mixin

The alt-copy() mixin can optionally be called with a font-size parameter. By default, it sets font-size to sm. It will automatically make the font responsive via a built-in call to the responsive-font() mixin.

// scss
.ticket-fees {
  @include alt-copy('md');
}

// output
.ticket-fees {
  font-size: 14px;
  font-weight: normal;
  color: #999;
}

@media (min-width: 45em) {
  .ticket-fees {
    font-size: 16px;
  }
}

Breakpoint mixin

You can specify media queries by using the breakpoint() mixin.

The mixin accepts two parameters, $min and $max values. By default, if only one parameter is passed, it is assigned to the $min value.

The mixin takes px values and converts them to em units, which is the preferred way to write media queries.

// scss
.some-block {
  margin-top: 15px;

  // $bp-md will be assigned as min-width
  @include breakpoint($bp-md) {
    margin-top: 40px;
  }
}

// output
.some-block {
  margin-top: 15px;
}

// note the conversion to em units
@media (min-width: 45em) {
  .some-block {
    margin-top: 40px;
  }
}

If you need to specify the maximum breakpoint instead, you call the mixin by explicity assigning the parameter to $max.

// scss
.another-block {
  [...]

  // here we are explicitly assigning our parameter to $max
  @include breakpoint($max: $bp-lg - 1) {
    padding: 25px;
  }
}

// output
[...]

@media (max-width: 63.6875em) {
  .another-block {
    padding: 25px;
  }
}

Note in the above example that we actually passed in ($max: $bp-lg - 1) because we want the styling rules to apply to anything below the max value. (We don't want to actually include the max value as part of the declaration.)

In our example, we're saying applying these rules to anything less than the 1020px width.

Where to Use breakpoint

In SASS, declare your breakpoints within the elements affected. Do not declare media queries as another code block separate from the original context.

// DON'T do this
.class1 {
  font-size: 12px;
}
.class2 {
  padding: 50px;
}

[...]

@include breakpoint($bp-md) {
  .class1 {
    font-size: 18px;
  }
  .class2 {
    padding: 100px;
  }
}

// DO do this
.class1 {
  font-size: 12px;

  @include breakpoint($bp-md) {
    font-size: 18px;
  }
}

Page Structure

The basic layout of our pages is as follows:

<body class="page edp">
  <div class="page__wrapper">
    <div class="page__frame">
      <div class="page__content">
        <!-- page content here -->
.page Sets base typography for page and background color
.page__wrapper Sets background color for a particular section of the page
.page__frame Sets max-width for your content area
.page__content Sets default font size and other needed attributes for your content area
.[pagename] Sets block context for any page-specific styles

Note that you don't always need to use all of the classes above. Only use those classes that you need to establish page structure.

For example, if you don't need set to a background color for your section of a page that's different from the body background, then you don't need to use page__wrapper. Similarly, if your content is supposed to span the full width of the browser at all times, then you don't need page__frame.

Content Spacing

All major page elements have standard vertical spacing that is applied as bottom margin of 20px on mobile and 40px on tablet/desktop.

Reference the content-spacing() mixin to apply the margins to your class.

// scss
.major-block {
  @include content-spacing();
}

// output
.major-block {
  margin-bottom: 20px;
}

@media (min-width: 45em) {
  margin-bottom: 40px;
}

The default is 10px of separation between these columns.

Radio Buttons

Radio buttons are styled by hiding the actual button and using CSS to create a visual replacement. (You can read more about the technique here.)

The structure of the HTML for radio buttons should be as follows:

<label class="radio-label" for="example-radio">
  <input class="radio-input" type="radio" id="example-radio" name="example-radio">
  <span>
    <span class="radio-label__name">
      Any Section
      <span class="radio-label__desc">
        (descriptive text)
      </span>
    </span>
  </span>
</label>

CSS Property Ordering

We currently utilize scss-lint to ensure consistency in our CSS.

One of the linters is configured to enforce a specific ordering and grouping of CSS properties. The ordering is primarily derived from Twitter's Recess project.
